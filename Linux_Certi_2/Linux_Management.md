## Chapter 01. 파일 시스템 관리
### 사용자 권한 및 그룹 설정
+ 모든 파일과 디렉터리에는 허가권(permission)과 소유권(ownership)이 부여된다.
+ 소유권 관련 명령어
  ```shell
  chown            // 📌 파일이나 디렉터리의 사용자 소유권과 그룹 소유권을 변경하는 명령어
  chgrp            // 파일이나 디렉터리의 그룹 소유권을 변경하는 명령어
  ```
  + 📌 ```chgrp -R kait data/```: data 디렉터리의 하위 디렉터리를 포함해서 디렉터리 내부의 모든 파일 및 디렉터리의 그룹 소유권을 kait로 변경하는 과정
+ 허가권 관련 명령어
  ```shell
  // 허가권은 읽기, 쓰기, 실행으로 구분
  chmod            // 📌 파일이나 디렉터리의 허가권을 변경하는 명령어
  umask            // 새로 생성되는 파일이나 디렉터리의 기본 허가권을 지정하는 명령어
  ```
+ 특수 허가권
  + 프로세스가 실행되는 동안만 해당 파일을 실행한 사용자가 아닌 해당 파일을 소유한 사용자의 권한으로 실행하도록 만드는 허가권
  + SetUID / SetGID / Sticky
+ 디스크 쿼터
  + 파일 시스템별 사용자나 그룹의 사용량을 제한할 경우에 사용
+ ```w```: 파일의 내용을 수정할 수 있는 권한
+ ```shell
  $ ls -l /user/bin/wall
  -r-xr-sr-x 1 root tty 15344 Jun 10 2022 /user/bin/wall
  // 📌 실행시킨 사용자는 일시적으로 tty 그룹 권한을 갖는다.
  ```
+ 📌 ```repquota```: /home 영역에 설정된 사용자 쿼터 정보를 확인하는 과정
### 파일 시스템 관리 유틸리티
+ 파일 시스템은 시스템에서 파일이나 자료를 쉽게 검색하고 접근, 유지, 관리하기 위한 조직체계이다.
  + ext, ext2, ext3, ext4
  + 저널링 파일 시스템, 네트워크 파일 시스템, 기타 파일 시스템
+ 파일 시스템 관련 명령어
  ```shell
  eject            // DVD나 CD-ROM 등의 미디어 장치를 해제하고 꺼내는 명령어
  df               // 시스템에 마운트된 하드디스크의 사용량과 남은 용량을 확인하는 명령어
  mount
  umount           // 📌 마운트 해제
  ```
+ 📌 ```/etc/fstab```: 파일에서 마운트되는 옵션 정보를 기록하는 필드 - 네 번째
+ 📌 원격지의 윈도우 시스템에 공유된 폴더를 마운트하는 과정 
  + ```mount -t cifs -o username=administrator, password=1234 //192.168.5.13/data/mnt```

## Chapter 02. Shell 관리
### 셸
+ 사용자가 입력한 명령어를 해석하여 커널에 전달한다.
+ ```$ echo $SHELL```: 환경변수를 이용해서 로그인한 셸을 확인하는 과정
+ 📌 ```set```: 선언된 셸 변수를 확인하는 명령
+ 📌 ```/etc/shells```: 현재 시스템에서 사용 가능한 셸의 정보를 저장하고 있는 파일
+ 📌 ```HISTSIZE```: 셸 실행 후 저장되는 history 개수를 확인할 수 있는 환경변수명
#### 본 셸 계열
+ 본 셸 bin/sh
  + 유닉스의 기본 셸로, Steve Bourne가 개발했다.
+ 콘 셸 bin/ksh
  + 본 셸의 확장 버전으로 David Korn이 개발했다. 
+ 배시 셸 bin/bash
  + 📌 본 셸을 기반으로 GNU 프로젝트에서 개발, 리눅스의 기본 셸
+ 지 셸 bin/zsh
  + 본 셸의 확장 버전으로, 콘 셸의 개선 버전이다.
  + 탭이나 방향키를 이용하여 선택 기능을 지원한다.
#### C 셸 계열
+ C 셸 bin/csh
  + Bill Joy가 개발, C언어 특징을 포함
+ TC 셸 bin/tcsh
  + C 셸의 확장 버전으로 Ken Greer가 개발
  + 사용자 로그인 상태, 터미널 모니터링 기능 지원
#### 셸의 기능
+ 📌 명령어 히스토리 기능
  + 기존에 실행한 명령들을 위/아래 방향키를 사용하여 검색 및 편집하여 특정 명령을 반복해서 수행할 수 있다.
+ ```/etc/shells```
  + 현재 사용 가능한 셸 목록 정보가 저장된 파일명
### 환경설정
+ 전역변수: 하위 셸에 상속 가능
+ 지역변수: 특정 사용자에게만 적용, 상속 불가능

## Chapter 03. 프로세스 관리
### 프로세스
+ 중앙처리장치를 할당받아 실행 중인 프로그램
+ 프로세서가 할당되는 실체로서, 디스패치가 가능한 단위
  + CentOS 7 버전 리눅스에서는 부팅을 시작하면 커널이 ```systemd``` (이)라는 최초의 프로세스를 발생시키고, 이후 시스템 운영에 필요한 데몬을 비롯한 다른 프로세스들은 ```fork``` 방식으로 ```systemd``` 프로세스의 자식 프로세스로 생성하게 된다.
#### 포그라운드
+ 사용자가 명령어를 입력하면 터미널에서 작업을 입력한 명령을 해석하여 실행하고, 결과를 화면에 출력
+ 화면에 수행 결과가 출력되는 일반적인 명력 실행방식
#### 백그라운드
+ 프로세스가 실행되는 동안 다른 프로세스 실행가능
+ 하나의 셸에서 여러 개의 다른 프로세스를 동시에 실행할 수 있는 방식
#### fork()
+ 새로운 프로세스를 위한 메모리를 할당하고, 기존 프로세스를 복제하는 방식
#### exec()
+ exec()에 의해 호출된 프로세스만 메모리에 남는다.
#### 데몬(daemon)
+ 리눅스 시스템 부팅시 실행되는 백그라운드 프로세스
+ 사용자의 요청을 기다리고 있다가 요청이 오면 적절히 대응하는 프로세스
#### 시그널
+ 운영체제에서 사용하는 제한된 형태의 프로세스 간 통신
+ 특정 이벤트가 발생했을 경우 프로세스에 전달하는 신호
  + 📌 ```15```:  [ctrl]+[c] 키 조합으로 발생
### 프로세스 관리 유틸리티
#### 프로세스 관련 명령어
  ```shell
  ps      // 프로세스 상태 출력
  ps -l   // 📌 백그라운드로 실행시킨 프로세스의 우선순위 값을 확인하는 명령
  pstree  // 실행중인 프로세스를 트리 구조로 출력
  top     // 프로세스 상태를 실시간으로 확인하거나 모니터링하는 명령어
  nohup   // 📌 프로세스 중단없이 백그라운드 작업을 수행할 수 있게 하는 명령어
  top     // 📌 프로세스 상태를 실시간으로 확인하거나 모니터링하는 명령어
  nohup   // 프로세스 중단없이 백그라운드 작업을 수행할 수 있게 하는 명령어
  jobs    // 백그라운드로 실행 중인 프로세스를 확인하는 명령어
  fg,bg   // 포그라운드와 백그라운드 작업을 전환할 수 있는 명령어
  nice    // 실행할 프로그램의 우선순위를 보고, 설정값을 확인하는 명령어
  renice    // 프로세스의 우선순위 값을 변경하는 명령어 
  ```
  kill %(작업번호)   // 📌 작업번호 백그라운드 프로세스 종료
  ```
  # find / -name `+.txt` > list &
  ```
  + 📌 제시된 명령을 백그라운드 프로세스로 실행하려고 할 때
  + 📌 ```nice --10 bash```: 프로세스 아이디가 1222인 bash 프로세스의 우선순위 변경, NI 값을 -10으로 변경
  + 📌 ```[ctrl]+[z]```: 현재 실행 중인 포어그라운드 프로세스의 작업을 백그라운드 프로세스로 전환하기 위해 사용하는 키 조합
